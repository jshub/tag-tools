New developments

After a relatively quiet three year period Microformats are again receiving attention, with new microformats being proposed, and Microsoft, Yahoo and Mozilla all announcing new initiatives.

Meanwhile Google have also been pushing for cross vendor standardization of metadata for some of its products to enable backend data-sharing.

Yahoo has placed microformats at the core of its Yahoo! Open Strategy (Y!OS)1 and SearchMonkey2 developer tools.

Browsers

Both Firefox 3.1 and IE8 are expected to features some native support for microformats, however the exact behavior and developer API it not yet know.

Extensions

Mozilla have long had a relationship with IBM researchers, with a number of Semantic and Microformat orientated extensions being contributed, along with core HTML support for Accessibility clients. The most feature rich is Operator3, on which future Firefox native support for microformats in extensions and web pages is planned.

More recently Mix Online (with Microsoft) have released Oomph4, an IE Toolbar plugin and standalone Javascript library (based on jQuery) for exposing microformats on sites. This is expected to support the release of IE8 and its native support for hSlices5 (an extension to hAtom) and possibly others.

Microformats

hProduct

Prototyped at BestBuy this has now become a Microformats.org proposal6, and is compatible and combinable with the existing hListing and hReview.

BestBuy have integrated it with their internal search (example) as well as their Giftag service and browser extension and they have committed to:

In the near future, Giftag will contribute and maintain an open source hProduct studio application built using jQuery…

This microformat is of particular interest for use in making recommendations.

hSlices

Still under development by Microsoft for IE8 the purpose of an hSlice is to allow for the subscribing to partial content in a page in a similar manner as for a RSS feed. As a result the hSlice microformat is based on hAtom and has provoked much discussion in the Microformats community about its implementation.
